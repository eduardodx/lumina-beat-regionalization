Adapter-Fusion Regionalization Blueprint
Regionalizing a nucleotide foundation model with synthetic ABraOM frequency post-training and ClinVar pathogenicity fine-tuning
Prepared for implementation planningDate: 2026-06-15
Core recommendation: Build a modular adapter-fusion system, not a single Brazil-specific checkpoint. Train population adapters on allele-frequency objectives, train a separate pathogenicity adapter on strong-evidence ClinVar, then train a fusion/calibration layer that composes molecular and population evidence without leaking submitter provenance into the model.

1. Goals, scope, and non-negotiable design principles
The objective is to regionalize a nucleotide-sequence foundation model for Brazilian clinical variant interpretation while preserving general molecular pathogenicity reasoning. The regionalization signal comes from ABraOM allele frequencies and optionally other population resources. The pathogenicity label signal comes from strong-evidence ClinVar. These two signals must remain conceptually and technically distinct.
A regionalized model should not learn the trivial rule “Brazil-associated equals benign.” It should learn that Brazilian population frequency is a population-prior evidence source that can change the clinical posterior for some variants, especially variants that are rare globally but present or enriched in ABraOM.
The best architecture is therefore modular: a frozen base model, one or more independently trained population adapters, a pathogenicity adapter, and a fusion/calibration component. This prevents destructive fine-tuning and makes future regionalizations incremental.
1.1 What “properly adapted” means
Molecular effect is preserved: severe sequence-level evidence, such as canonical splice disruption or protein-truncating effect in an appropriate disease gene, should not disappear merely because a regional adapter is active.
Brazilian frequency evidence is used: variants that are common or enriched in ABraOM should be less likely to be called pathogenic for rare high-penetrance Mendelian disease, subject to disease architecture, inheritance mode, penetrance, and founder exceptions.
Calibration improves: the predicted probability should be better calibrated on Brazilian-submitted, Brazilian-enriched, and ABraOM-present strata.
False positives fall: the model should reduce false pathogenic calls among ABraOM variants that are too common to be plausible high-penetrance pathogenic variants.
Sensitivity is preserved: true P/LP variants, including Brazilian founder or ABraOM-present pathogenic variants, should not be suppressed.
Global performance is non-inferior: the model must not meaningfully degrade on non-Brazilian ClinVar temporal/gene/submitter holdouts.

Strong position: The production model should probably include explicit ABraOM/gnomAD frequency features in addition to population adapters. Adapter fusion can encode regional sequence-distribution priors, but explicit allele frequency remains too clinically informative to hide inside embeddings only.
2. Data roles and leakage boundaries
Regionalization fails when data sources are assigned the wrong semantic role. ABraOM should not be treated as a pathogenicity-label dataset. ClinVar should not be treated as a population-frequency dataset. Brazilian-submitted ClinVar records should not be treated as proof that the variant arose in a Brazilian patient unless that is explicitly documented in the submission or linked evidence.

2.2 ABraOM assumptions
The ABraOM WGS dataset is based on 1,171 unrelated SABE participants from São Paulo, uses hg38 for the WGS release, and contains tens of millions of SNVs and short indels. The cohort is elderly and admixed. That makes it valuable for identifying variants that are locally common but underrepresented globally, but it is not a complete representation of all Brazilian populations.
The practical consequence is asymmetric evidence: a variant being common in ABraOM can be strong benign population evidence for many rare Mendelian contexts; a variant being absent from ABraOM is generally weak evidence because the cohort size is modest relative to rare-disease allele frequencies.
2.3 ClinVar assumptions
ClinVar stores submitted classifications as SCV records, aggregates by variant-condition as RCV records, and aggregates by variant as VCV records. For this project, preserve SCV-level provenance even if the final label is VCV-level or variant-level, because you need submitter country/lab metadata for splitting and leakage control.
Use review status and strong-evidence filtering for inclusion and weighting. Do not use review status, submitter country, or submitter identity as inference-time model features.

3. Synthetic ABraOM frequency post-training
The purpose of ABraOM post-training is to create a Brazilian population adapter A_BR that makes the base model more aware of Brazilian allele distributions. It is not meant to teach pathogenicity. The best objective uses ABraOM allele counts as a soft target for allele probabilities at known variant sites.
3.1 Core frequency objective for SNVs
For a biallelic SNV v at locus i:
reference allele = REF_ialternate allele = ALT_iABraOM allele count = AC_iABraOM allele number = AN_iSmoothed Brazilian alternate-allele target:  f_BR_i = (AC_i + alpha) / (AN_i + alpha + beta)Jeffreys-style default:  alpha = 0.5  beta  = 0.5
Train the adapter to predict the observed reference/alternate allele distribution under sequence context:
L_ABraOM =  - sum_i w_i [      AC_i        * log P_theta(ALT_i | context_i)    + (AN_i-AC_i) * log P_theta(REF_i | context_i)  ]Equivalent normalized form:  L_ABraOM =  - sum_i w_i [      f_BR_i       * log p_alt_i    + (1-f_BR_i)   * log (1-p_alt_i)  ]
The weight w_i should reflect call confidence, allele number, region callability, mapping ambiguity, and desired AF-bin balancing. Without balancing, very common and very rare bins can dominate training in undesirable ways.
3.2 Sampling strategy
Use sparse perturbations: If you only have aggregate frequencies, avoid building fully synthetic genomes by independently sampling all variants. That destroys haplotypes, LD, phase, and local ancestry.
Use single-locus or sparse multi-locus contexts: For each training example, generate ref and alt contexts around one or a few variants. Keep the reference genome context otherwise intact.
Balance by AF bin: Sample from bins such as singleton/very rare, low-frequency, moderate-frequency, common, and Brazil-enriched. Do not allow common variants alone to define the adapter.
Hold out variants: Create ABraOM_train_freq and ABraOM_test_freq before post-training. Also create ABraOM_test_no_ClinVar_overlap for strict evaluation.
Treat indels separately: For indels, use ref/alt sequence pseudo-likelihood ratio or span-level reconstruction rather than forcing a single-token allele prediction.

3.4 ABraOM post-training validation
Before touching ClinVar, prove that A_BR actually learned Brazilian frequency structure. Use held-out ABraOM variants and compare the frozen base model against the base plus A_BR.
Frequency negative log-likelihood: binomial cross-entropy between predicted p_alt and observed ABraOM AF.
Brier score: squared error between predicted p_alt and f_BR.
Spearman correlation: correlation between ref/alt log-likelihood ratio and logit(AF_BR).
Calibration by AF bin: predicted versus observed AF in bins: singleton/rare, 0.01–0.1%, 0.1–0.5%, 0.5–1%, 1–5%, >5%.
Brazil-enriched ranking: variants common in ABraOM but rare globally should rank higher under A_BR than under global-only adapters.
Controls: scrambled ABraOM adapter should not improve these metrics; global-frequency adapter may improve generic frequency prediction but should not fully explain Brazil-enriched gains.
4. Adapter-fusion architecture
Adapter fusion allows regionalization to scale. Instead of creating one full model checkpoint per country or cohort, keep a frozen base model and compose small adapters through a learned fusion controller. This is directly inspired by the AdapterFusion idea of separating knowledge extraction from knowledge composition.
4.1 Layer-wise LoRA fusion
For a transformer layer with base projection W, a LoRA adapter adds a low-rank update ΔW = B A. A regional fusion layer uses a weighted sum of regional low-rank updates:
Base layer:  y = W xSingle LoRA adapter:  y = W x + DeltaW_path xFused adapters:  y = W x      + DeltaW_path x      + sum_r alpha_{layer,r}(x, meta) * DeltaW_r x
Here r indexes population adapters such as A_BR, A_global_pop, A_AFR, A_EUR, A_AMR, A_EAS, and A_SAS. The fusion weights alpha may be fixed, metadata-driven, or learned dynamically.

4.5 Recommended fusion block pseudocode
class RegionalFusionBlock(nn.Module):    def __init__(self, base_block, path_adapter, region_adapters, hidden_dim, meta_dim):        super().__init__()        self.base_block = freeze(base_block)        self.path_adapter = path_adapter        self.region_adapters = nn.ModuleDict(region_adapters)        self.gate = nn.Sequential(            nn.Linear(hidden_dim + meta_dim, hidden_dim // 4),            nn.GELU(),            nn.Linear(hidden_dim // 4, len(region_adapters))        )    def forward(self, h, meta):        base_out = self.base_block(h)        path_update = self.path_adapter(h)        pooled = pool_variant_center(h)        alpha = torch.softmax(self.gate(torch.cat([pooled, meta], dim=-1)), dim=-1)        region_update = 0        for k, adapter in enumerate(self.region_adapters.values()):            region_update = region_update + alpha[:, k, None, None] * adapter(h)        return base_out + path_update + region_update, alpha
4.6 Regularization
Adapter dropout: randomly drop one regional adapter during fusion training so the controller cannot rely on a single adapter shortcut.
Region dropout: hide explicit ABraOM/gnomAD features on a fraction of batches to force robustness.
Sparsity or entropy control: prevent either total collapse to A_BR or noisy use of all adapters everywhere.
Counterfactual consistency: changing target region should not drastically change a score unless allele-frequency evidence differs materially.
Monotonic frequency constraint: for rare high-penetrance Mendelian disease, increasing credible regional AF should not increase pathogenic probability when other evidence is fixed.
Adapter norm penalty: prevent regional updates from overwhelming the pathogenicity adapter.
5. Training protocol
Training should occur in separated phases. The point is to let each adapter acquire one kind of knowledge, then learn composition. Do not fine-tune the entire system end-to-end from the beginning.
5.1 Phase 0 — normalization and dataset build
Normalize all variants to GRCh38 chr:pos:ref:alt. Left-normalize, split multiallelic variants, and harmonize representation across ABraOM, ClinVar, and gnomAD.
Preserve original identifiers: ClinVar VariationID, VCV, RCV, SCV list, submitters, conditions, review status, last-evaluated date, ABraOM AC/AN/AF, and gnomAD AC/AN/AF.
Build variant-level and optionally variant-condition-level tables. Do not collapse variant-condition records if disease context is central to your task.
Create leakage-resistant split keys: normalized_variant_id, gene, condition/disease group, submitter, codon/amino-acid change, splice motif, genomic position.
5.2 Phase 1 — train population adapters
for region in population_regions:    freeze(F0)    train(A_region) on frequency objective for that region    validate(A_region) on held-out frequency sites    save adapter checkpoint and frequency-alignment report
For the first build, train at least A_BR and one comparator adapter A_global or A_gnomAD. If you later add LATAM, Japan, Finland, or other cohorts, train them independently and plug them into the same fusion bank.
5.3 Phase 2 — train the pathogenicity adapter
Train A_path and an initial pathogenicity head on strong-evidence ClinVar. Use paired reference and alternate sequence contexts.
Input per variant:  ref_context = reference sequence around variant  alt_context = same sequence with REF replaced by ALTRepresentations:  h_ref = F0 + A_path(ref_context)  h_alt = F0 + A_path(alt_context)  delta = h_alt - h_refHead input:  z = [pool(h_ref), pool(h_alt), pool(delta), optional likelihood-ratio features]Target:  P/LP = 1  B/LB = 0
For the cleanest regionalization experiment, train A_path on nonBR_only ClinVar and hold out Brazilian-submitted variants. This tests whether ABraOM adaptation helps without supervised Brazilian ClinVar labels.
5.4 Phase 3 — train fusion and regional head
Freeze F0 and the independently trained adapters. Train only the fusion gates and regional head first. This mirrors the knowledge-composition stage from AdapterFusion.
Frozen:  F0  A_path  A_BR  A_global / A_gnomAD / ancestry adaptersTrainable:  G         = fusion gates  H_reg     = regional pathogenicity/calibration head  optional H_mol calibration layerLoss:  L_total =      L_ClinVar    + lambda_brier * L_Brier    + lambda_freq  * L_frequency_consistency    + lambda_gate  * L_gate_regularization    + lambda_mono  * L_monotonic_frequency_penalty
A practical loss starts simple: weighted BCE for ClinVar plus a mild calibration loss or post-hoc calibration. Add monotonic penalties only after you have a stable baseline.
5.5 Phase 4 — optional supervised Brazilian fusion
After the clean zero-shot/leave-Brazilian-out experiment, you may train with Brazilian-submitted ClinVar. Then evaluation must be stricter: leave-Brazilian-submitter-out, temporal holdout, gene holdout, and variant-family holdout. Do not train and test on different SCVs of the same variant.

This is the safety counterpart to the pseudo-benign benchmark. It detects whether regionalization learned an overbroad benign prior.

Brazilian-submitted P/LP variants held out from training.
Curated Brazilian founder/pathogenic variants from independent sources.
P/LP variants in recessive genes where nonzero carrier frequency is plausible.
Severe molecular-effect variants where population evidence should not dominate without disease context.
Metrics: sensitivity, false-benign rate, score drop from non-regionalized to regionalized model, and number of P/LP variants crossing below the reporting threshold.

7.8 Statistical testing
Clustered bootstrap: resample by gene, disease group, or gene+submitter rather than independent variants. Report confidence intervals for metric differences.
Paired comparisons: compare M4 versus M0 on the same test variants; report ΔAUPRC, ΔBrier, ΔECE, Δfalse-pathogenic-rate, and Δsensitivity.
Permutation sanity check: shuffled ABraOM labels or randomized adapter assignment should not yield the same gain.
Pre-register operating points: choose sensitivity-at-specificity or specificity-at-sensitivity thresholds before looking at the final test set.

8.4 What would falsify the approach
A_BR improves neither held-out ABraOM frequency prediction nor Brazil-enriched ranking.
M4 gains are matched by scrambled A_BR.
M6, the explicit-frequency-only model, clearly beats M5 with lower complexity.
False pathogenic calls fall only because sensitivity on P/LP variants collapses.
Performance improves only on random ClinVar splits but not temporal, gene, submitter, or variant-family splits.
The gate always selects A_BR whenever target_region=Brazil regardless of AF, consequence, and molecular evidence.
9. Implementation blueprint
This section turns the design into build tasks.
9.1 Minimum viable build
Normalize variants: create one GRCh38 left-normalized table joining ClinVar, ABraOM, and gnomAD.
Create ClinVar provenance flags: nonBR_only, BR_only, mixed_BR_nonBR, BR_any.
Create ABraOM frequency splits: ABraOM_train_freq, ABraOM_test_freq, ABraOM_test_no_ClinVar_overlap.
Train A_BR: use the ABraOM frequency objective with balanced AF-bin sampling.
Validate A_BR: frequency NLL/Brier/correlation/calibration on held-out ABraOM.
Train M0: F0 + A_path on nonBR_only ClinVar.
Train M4: frozen F0 + A_path + A_BR + global adapter with learned fusion.
Train M5: M4 plus explicit ABraOM/gnomAD frequency features.
Evaluate: nonBR holdout, BR_only, mixed, Brazil-enriched ClinVar, ABraOM pseudo-benign, P/LP sensitivity protection.
Report ablations: include M2, M6, and scrambled A_BR at minimum.

This output makes error review easier. If a variant is downweighted in Brazil because ABraOM AF is high, the system should expose that explicitly.

10. Production and monitoring notes
The deployment risk is not that the adapter fusion is too weak; it is that the model becomes overconfident and hides the distinction between molecular effect and population evidence. The production interface should preserve this distinction.
10.1 Recommended production modes
Mode
Use
Caution
Research prioritization
High sensitivity; low threshold; returns ranked candidates.
Accepts more false positives.
Clinical triage
Moderate threshold; requires evidence decomposition.
Should avoid automatic pathogenic assertions.
Benign suppression
High-specificity benign downranking of common regional variants.
Requires strong calibration and false-negative protection.
Patient-level diagnosis support
Variant ranking with phenotype, inheritance, and family segregation if available.
Requires separate patient-level validation.
10.2 Monitoring
Track score drift after each ClinVar or ABraOM update.
Track false pathogenic rate by ABraOM AF bin and consequence class.
Track sensitivity on a frozen P/LP sentinel panel, including Brazilian founder/pathogenic variants.
Track gate behavior: mean alpha_BR by AF bin should not change unexpectedly between releases.
Re-run temporal ClinVar evaluation after each major release.
Keep adapter versions independent so A_BR can be updated without retraining F0 or A_path from scratch.
10.3 Future regionalizations
The same architecture allows new regional adapters. For a new region R, train A_R on the region’s allele-frequency or haplotype data, validate frequency alignment, plug A_R into the adapter bank, then evaluate region-enriched ClinVar/local clinical variants and common-region pseudo-benign benchmarks. The base model and A_path do not need to be retrained unless the task definition changes.
Add a new region:  1. Normalize regional frequency table to GRCh38.  2. Create train/test frequency splits.  3. Train A_region with same objective as A_BR.  4. Validate frequency alignment.  5. Add A_region to fusion bank.  6. Calibrate if local labels exist.  7. Evaluate:       region-enriched ClinVar       region common pseudo-benign       local P/LP sensitivity panel       global non-inferiority
10.4 Practical recommendation
Build order: First build M0, A_BR, M4, M5, M6, and scrambled A_BR. Do not invest in token-wise fusion, end-to-end fine-tuning, or many regional adapters until M5 beats M0/M2/M6 on Brazil-enriched calibration and false-positive reduction while preserving P/LP sensitivity.
